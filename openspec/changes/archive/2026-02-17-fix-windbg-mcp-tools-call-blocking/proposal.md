## 为什么

当前 WinDbg 窗口中的 MCP 调试输出以单行摘要为主，遇到阻塞、超时或调用链异常时，信息不够直观，定位成本高。现在需要提供更可读、可关联、可排障的日志链路，以缩短问题发现与归因时间。

## 变更内容

- 优化 WinDbg 窗口中的调试日志格式，使请求生命周期（接收、路由、工具执行、返回）更清晰可读。
- 补充排障关键字段（如请求标识、方法名、工具名、阶段、耗时、结果状态、错误摘要），支持快速关联同一请求的多条日志。
- 强化异常路径可观测性，明确标识执行阻塞/失败场景的上下文信息，避免仅看到“收到请求”却无法判断卡在哪个阶段。
- 保持对外 MCP HTTP 接口与工具契约不变，不引入 BREAKING 变更。

## 功能 (Capabilities)

### 新增功能

- `windbg-mcp-debug-observability`: 为 WinDbg 侧 MCP 调试链路提供结构化、可关联、面向排障的日志输出能力。

### 修改功能

- `windbg-http-mcp-eval`: 扩展现有规范中的日志可观测性要求，明确请求阶段日志、关键字段、异常路径与可读性标准。

## 影响

- 代码范围：`src/dbgx-mcp.cpp`、`src/mcp/io_echo.cpp`、可能涉及 `src/mcp/json_rpc.cpp` 与命令执行链路日志打点位置。
- 测试范围：更新/新增与日志输出格式、字段完整性、敏感信息脱敏、异常路径覆盖相关的单元测试。
- 文档范围：README 中的诊断与示例日志片段可能需要同步更新。
- 运行影响：主要是日志内容与格式调整，预计无对外 API 破坏。
