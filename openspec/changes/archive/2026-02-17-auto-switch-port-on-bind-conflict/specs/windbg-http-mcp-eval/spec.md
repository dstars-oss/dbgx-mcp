## 新增需求

### 需求:MCP HTTP 服务启动必须支持端口冲突自动回退
系统在启动 HTTP 监听时，必须先尝试配置或默认端口；当该端口被占用时，必须按预定义顺序尝试后续候选端口，直到绑定成功或达到最大尝试次数。系统必须在 WinDbg 可见输出中记录冲突重试与最终监听结果。

#### 场景:初始端口可用时直接绑定
- **当** 服务启动时初始端口可成功绑定
- **那么** 系统必须在初始端口完成监听并进入可服务状态
- **并且** 系统必须输出最终监听端口且该端口等于初始端口

#### 场景:初始端口占用后自动回退成功
- **当** 初始端口被其他进程占用且候选端口范围内存在可用端口
- **那么** 系统必须按既定顺序继续尝试后续端口并在首个可用端口绑定成功
- **并且** 系统必须输出端口冲突信息与最终绑定端口
- **并且** `/mcp` 端点必须在最终绑定端口可用

#### 场景:连续冲突达到上限后启动失败
- **当** 候选端口在最大尝试次数内均不可用
- **那么** 系统必须返回明确的启动失败结果并输出尝试次数与失败摘要
- **并且** 系统禁止进入可服务状态

#### 场景:非端口占用错误快速失败
- **当** 监听绑定因非“端口占用”错误失败（如权限不足或地址无效）
- **那么** 系统必须立即失败并输出不可重试错误摘要
- **并且** 系统禁止继续尝试后续端口
