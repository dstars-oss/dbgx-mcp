## 为什么

当前 `README` 更偏向实现与维护者视角，虽然覆盖了构建、加载与调试命令，但对首次接触项目的使用者来说，缺少“这是什么、我为什么要用、我该先做什么”的清晰路径。项目已具备 MVP 可用性，现在需要降低上手门槛，提升试用与验证效率。

## 变更内容

- 重构 `README` 信息结构，优先呈现使用者最关心的信息：项目价值、适用人群、使用场景与快速上手入口。
- 新增“最小可用路径（Quick Start）”，用一步一步的方式串联从前置条件、加载扩展到首次 MCP 调用的完整流程。
- 将现有偏开发者维护的信息（实现目标、测试策略、深度排障）下沉到后置章节，避免干扰首次使用流程。
- 统一中英文文档的核心信息结构，确保不同语言读者获得一致的使用体验。

## 功能 (Capabilities)

### 新增功能
- 无

### 修改功能
- `windbg-http-mcp-eval`: 扩展用户文档要求，从“提供可执行加载说明”提升为“提供以使用者为中心的项目介绍与上手流程”，明确包含价值说明、使用前提、最小验证路径与常见问题入口。

## 影响

- 受影响文件：`README.md`、`README.zh-CN.md`（如存在并需同步）。
- 受影响规范：`openspec/specs/windbg-http-mcp-eval/spec.md`（后续在 specs 产出物中补充/调整文档相关需求）。
- 对 API、二进制接口、构建流程与运行时行为无破坏性影响。
