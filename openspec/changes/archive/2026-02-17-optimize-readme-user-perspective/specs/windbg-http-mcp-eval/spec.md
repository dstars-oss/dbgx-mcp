## 修改需求

### 需求:加载说明必须提供可执行诊断路径
系统必须在用户文档中提供以使用者为中心的项目介绍、可直接执行的 `.load` 示例、最小可用验证路径与常见故障诊断步骤，且不得要求读者先理解实现细节后才能完成首次使用。

#### 场景:使用者可快速理解项目价值
- **当** 使用者首次打开项目 README
- **那么** 文档必须在前置条件和构建说明之前给出项目用途、适用人群与典型使用场景
- **并且** 该介绍必须让读者明确“为何使用本项目”与“何时使用本项目”

#### 场景:最小上手路径可直接执行
- **当** 使用者按照文档执行 Quick Start
- **那么** 文档必须提供从前置条件、扩展加载到首次 MCP 请求验证的顺序化步骤
- **并且** 使用者必须能够在不阅读源码的情况下完成至少一次成功的 `initialize` 或 `tools/call`

#### 场景:路径写法可直接复用
- **当** 使用者按照文档使用绝对路径加载扩展
- **那么** 文档必须给出不会导致路径分隔符歧义的命令写法示例

#### 场景:加载失败有诊断指引
- **当** `.load` 失败并返回错误码
- **那么** 文档必须提供对应的最小排查步骤（路径存在性、依赖检查或导出检查）

#### 场景:中英文文档结构保持一致
- **当** 项目同时提供 `README.md` 与 `README.zh-CN.md`
- **那么** 两份文档必须包含一致的核心章节（项目概览、前置条件、Quick Start、故障诊断入口）
